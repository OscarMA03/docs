## Push Zipcodes

Push zipcodes for a company/class to define service areas.

### Headers

<ParamField path="header" type="string" name="x-api-key" required>
  API key for authentication
</ParamField>

<ParamField path="header" type="string" name="Content-Type" required>
  application/json
</ParamField>

### Request Body

<ParamField path="body" type="array" name="zipcodes" required>
  Array of ZIP code strings to add
</ParamField>

<ParamField path="body" type="number" name="class_id" required>
  Class identifier
</ParamField>

<ParamField path="body" type="number" name="company_id">
  Company identifier (optional)
</ParamField>

<ParamField path="body" type="string" name="county">
  County name (optional)
</ParamField>

<ParamField path="body" type="string" name="state" required>
  State abbreviation or name
</ParamField>

### Example Request

```bash
POST /push-zipcodes
Content-Type: application/json
x-api-key: <your-api-key>

{
  "zipcodes": ["10001", "10002", "10003"],
  "class_id": 123,
  "company_id": 456,
  "county": "New York County",
  "state": "NY"
}
```

### Example Response

```json
{
  "success": true,
  "message": "Zipcodes pushed successfully",
  "data": {
    "class_id": 123,
    "zipcodes_added": 3
  }
}
```

### Error Response

```json
{
  "success": false,
  "error": "Invalid class_id or missing required fields"
}
```

### Response Fields

<ParamField path="response" type="boolean" name="success">
  Indicates if the request was successful
</ParamField>

<ParamField path="response" type="string" name="message">
  Success message (if successful)
</ParamField>

<ParamField path="response" type="object" name="data">
  Response data object
</ParamField>

<ParamField path="response" type="number" name="data.class_id">
  The class identifier that was updated
</ParamField>

<ParamField path="response" type="number" name="data.zipcodes_added">

</ParamField>

<ParamField path="response" type="string" name="error">
  Error message (if failed)
</ParamField>

### Use Cases

- **Service Area Definition**: Define which zipcodes are covered by a company/class
- **Bulk Updates**: Add multiple zipcodes at once for a service area
- **Geographic Configuration**: Configure service areas for different regions
- **Location Management**: Manage service area coverage by zipcode

### Implementation Notes

- `zipcodes` must be an array of strings
- `class_id` and `state` are required fields
- `company_id` and `county` are optional
- All zipcodes in the array will be added to the specified class
- Requires `x-api-key` header for authentication
