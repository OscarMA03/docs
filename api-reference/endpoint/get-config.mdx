## Get Company Configuration

Retrieves company configuration settings by company ID.

<ParamField path="query" type="string" name="companyID" required>
  Unique identifier for the company
</ParamField>

<ParamField path="header" type="string" name="X-API-Key" required>
  API key for authentication
</ParamField>

### Example Request

```bash
GET /api/config?companyID=123
X-API-Key: your_api_key_here
```

### Example Response

```json
{
  "success": true,
  "data": {
    "companyID": 123,
    "companyUrl": "https://company-api.example.com",
    "companyName": "Example Company Inc",
    "companyNumber": "+1234567890",
    "serviceLabelType": "appointment",
    "aiDisclosure": true,
```

### Error Response

```json
{
  "success": false,
  "data": null,
  "error": "Company configuration not found"
}
```

### Use Cases

- **AI Agent Initialization**: Retrieve company-specific settings for AI agent setup
- **Configuration Validation**: Verify existing configuration before updates
- **System Integration**: Access configuration data for third-party integrations
- **Audit Trail**: Review current configuration settings

### Implementation Notes

- Each agent should have a hardcoded `companyId` value when hitting this endpoint
- Configuration data should be cached for performance
- Consider implementing versioning for configuration changes
- Ensure proper authentication and authorization for sensitive configuration data
