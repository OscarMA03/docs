## Get Agent Configuration

Retrieves agent configuration for an agent by LLM ID.

### Authentication

<ParamField path="header" type="string" name="X-API-Key" required>
  API key for authentication
</ParamField>

### Request Parameters

<ParamField path="query" type="string" name="llm_id" required>
  Retell LLM identifier
</ParamField>

### Example Request

```bash
GET /get-config?llm_id=123
X-API-Key: your_api_key_here
```

### Example Response

```json
{
  "success": true,
  "data": {
    "configuration": {
      "customGreeting": "Hi, thanks for calling Example Company",
      "companyName": "Example Company Inc",
      "businessHours": "Monday 9am-5pm, Friday 9am-5pm",
      "servicesSupported": "windows, doors, replacements",
      "agentName": "Sally",
      "companyUrl": "https://company-api.example.com",
      "aiDisclosure": "true",
      "servicesNotIncluded": "repairs, roofs",
      "companyID": "123",
      "companyNumber": "+1234567890",
      "serviceLabelType": "In Home Consultation",
      "callback": "callback_task",
      "serviceAreas": "New York, New Jersey"
    }
  },
  "error": null
}
```

### Error Response

```json
{
  "success": false,
  "data": null,
  "error": "LLM configuration not found"
}
```

### Use Cases

- **Retrieve Agent Settings**: Get all configuration variables for a Retell LLM agent
- **Configuration Verification**: Verify current agent configuration settings
- **System Integration**: Access agent configuration for third-party integrations
- **Configuration Management**: Review and manage agent configurations

### Implementation Notes

- Returns an object containing `configuration` with all agent configuration variables
- All configuration variables are nested within the `configuration` object
- Configuration data is specific to the LLM ID provided
- Ensure proper authentication and authorization for accessing agent configurations
