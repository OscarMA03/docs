## Update Agent Configuration

Updates agent configuration settings for an existing agent by LLM ID.

### Authentication

<ParamField path="header" type="string" name="X-API-Key" required>
  API key for authentication
</ParamField>

### Path Parameters

<ParamField path="path" type="string" name="llm_id" required>
  Retell LLM identifier for the agent to update
</ParamField>

### Request Body

<ParamField path="body" type="object" name="configData" required>
  Company configuration data to update
</ParamField>

<ParamField path="body" type="string" name="classId">
  Class identifier
</ParamField>

<ParamField path="body" type="string" name="companyUrl">
  Company's API endpoint
</ParamField>

<ParamField path="body" type="string" name="companyName">
  Official business name
</ParamField>

<ParamField path="body" type="string" name="companyNumber">
  Callback number for follow-ups
</ParamField>

<ParamField path="body" type="string" name="serviceLabelType">
  What to call the appointment (e.g., "In Home Consultation")
</ParamField>

<ParamField path="body" type="string" name="aiDisclosure">
  Whether agent announces it's AI (e.g., "true")
</ParamField>

<ParamField path="body" type="string" name="businessHours">
  When to offer appointments
</ParamField>

<ParamField path="body" type="string" name="serviceAreas">
  Geographic restrictions

<ParamField path="body" type="string" name="callback">
  Create task to callback at later time
</ParamField>

<ParamField path="body" type="string" name="customGreeting">

<ParamField path="body" type="string" name="servicesNotIncluded">
  Comma-separated list of services excluded (e.g., "repairs, roofs")
</ParamField>

### Use Cases

- **Configuration Updates**: Update existing agent configuration settings
- **Business Rule Changes**: Modify appointment types, business hours, or service areas
- **Agent Customization**: Update agent name, custom greeting, or disclosure settings
- **Service Management**: Modify supported or excluded services

### Implementation Notes

- Updates configuration for an existing agent identified by the LLM ID in the path
- Only fields included in the request body will be updated (partial updates are supported)
- Fields not included in the request will remain unchanged
- Returns the updated configuration object in the response
